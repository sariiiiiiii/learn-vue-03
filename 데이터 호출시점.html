[데이터 호출 시점] ** 라우터 네이게이션 가드가 라이프 사이클 훅보다 먼저 실행된다 ** 컴포넌트가 생성되고 나서 호출할건지 아니면 라우팅 상태에서 데이터를 호출할건지 선택 1. 라우터 네비게이션 가드 - 라우터로 특정 URL에 접근했을 때 접근하기전에 동작들을 정의하는 속성(함수) // router index.js // beforeEnter는 보통 인증 정보를 검열할 때 사용 beforeEnter: (to, from, next) => { to: 이동할 URL의 라우팅 정보 from: 현재 URL의 라우팅 정보 next(); 를 해줘야지만 이동한다 } 2. 컴포넌트 라이프 사이클 훅 - created : 컴포넌트가 생성되자마자 호출되는 로직 (화면에 인스턴스의 내용들이 붙어있는 상태는 아님) - NewsView, AskView, JobsView 처럼 컴포넌트를 나눠서 하게 되면 상태관리를 공유하고 있지 않기 때문에 상관이 없는데 ListView처럼 상태관리를 공유하게 되면 데이터가 늦게 처리 되는 것을 볼 수 있다
