[데이터 호출 시점] ** 라우터 네이게이션 가드가 라이프 사이클 훅보다 먼저 실행된다 ** 컴포넌트가 생성되고 나서 호출할건지 아니면 라우팅 상태에서 데이터를 호출할건지 선택 1. 라우터 네비게이션 가드 - 라우터로 특정 URL에 접근했을 때 접근하기전에 동작들을 정의하는 속성(함수) // router index.js // beforeEnter는 보통 인증 정보를 검열할 때 사용 beforeEnter: (to, from, next) => { to: 이동할 URL의 라우팅 정보 from: 현재 URL의 라우팅 정보 next(); 를 해줘야지만 이동한다 } router index.js에서 컴포넌트 이동전에 created로 데이터를 미리 받고 스피너 endpoint를 데이터를 받는 즉시가 아닌 컴포넌트가 mounted되는 시점으로 적용 NewsView, AskView, JobsView가 동일하게 mounted를 쓰고 있기 때문에 그 코드를 mixin으로 따로 뺀 작업 ** 라우터 네비게이션 가드를 쓰게 되면 데이터가 불러와지는 과정에서 기다리게 되는 시점이 생기게 될 경우 사용자에게 기다리라는 기타 도구 등이 필요하다(UI COMPONENT) 스피너, 로딩 등등.... 2. 컴포넌트 라이프 사이클 훅 - created : 컴포넌트가 생성되자마자 호출되는 로직 (화면에 인스턴스의 내용들이 붙어있는 상태는 아님) - NewsView, AskView, JobsView 처럼 컴포넌트를 나눠서 하게 되면 상태관리를 공유하고 있지 않기 때문에 상관이 없는데 ListView처럼 상태관리를 공유하게 되면 데이터가 늦게 처리 되는 것을 볼 수 있다
